# Markdown Mastery Skill

## Description

Expertise in generating and refactoring Markdown documentation following strict CommonMark and GitHub Flavored Markdown (GFM) standards.

## Strict Syntax Rules

- **Headings:** Always use ATX-style (`#`). Mandatory space after `#`.
- **Lists:** Use hyphens `-` for unordered lists. 2-space indentation for nesting.
- **Code Blocks:** Mandatory language identifier. Use fenced blocks (```) only.
- **Tables:** Must include header delimiters `|---|` and be visually aligned in source.
- **Links:** Use descriptive text. No raw URLs; wrap them in `< >` or use `[text](url)`.

## Structural Requirements

- **One H1 Per File:** Every file must start with exactly one level 1 heading.
- **Hierarchy:** No skipping heading levels (e.g., `#` to `###` is forbidden).
- **Whitespace:** Single blank line between blocks (paragraphs, lists, code).
- **Line Breaks:** Use double-space or backslash for manual line breaks within a paragraph.

## Formatting Standards

- **Emphasis:** `**bold**`, `*italic*`, `~~strikethrough~~`.
- **Assets:** Images must have alt-text: `![alt](url)`.
- **Checklists:** Use `- [ ]` and `- [x]` for task tracking.
