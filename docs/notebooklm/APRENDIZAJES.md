# APRENDIZAJES — TheForge

Conocimiento y experiencia del proyecto para traspaso a juniors: qué pedimos, cómo lo pedimos, errores, pivotes y tiempos. Actualizar con cada sesión relevante.

---

## 1. Propósito y audiencia

- **Para quién:** Juniors y onboarding al repo TheForge.
- **Cómo usarlo:** Leer antes de tocar flujos MDD, DBGA o Workshop. Consultar "Errores y soluciones" y "Pivotes" cuando algo falle o sea confuso.

---

## 2. Qué pedimos (objetivos del producto)

- **TheForge:** Orquestador de "software factory" — entrevista proactiva (IA) → MDD → Semáforo → estimación MXN → entregables.
- **Fase 0:** Benchmark & Gap Analysis (DBGA) con opción de URLs, scraping (Cheerio), deep research (endpoint separado).
- **MDD:** Documento maestro estructurado (schema Zod) con **7 secciones canónicas:** 1. Contexto y Alcance, 2. Arquitectura y Stack, 3. Modelo de Datos, 4. Contratos de API, 5. Lógica y Edge Cases, 6. Seguridad, 7. Infraestructura. Generado por agentes (Manager, Clarifier, Software Architect, Security, Integration, Formatter, Diagram Injector, Auditor); markdown derivado con `mddStructuredToMarkdown`. **Matriz de delegación (sin traslape):** Clarifier→1, Software Architect→2,3,4,5, Security→6, Integration→7 (la §3 la redacta el Arquitecto de Software, no un prompt `data-model-expert` separado). **Criterios por sección:** El Auditor evalúa las 7 con reglas mínimas por sección (Fronteras/Audiencia en 1, versiones/patrones/Metadata en 2, Paridad SQL-Mermaid/UUID/TIMESTAMPTZ en 3, /health y payloads/códigos en 4, Flujos maestros/excepciones en 5, Sustento/Super Admin/security_events en 6, env vars y CI/CD en 7); cada agente tiene en su prompt solo las reglas de su sección.
- **Workshop:** Vista 3 columnas (chat, MDD viewer, Semáforo + costos); estado Zustand; API `POST /ai-orchestrator/chat` y streams de ai-analysis.
- **Etapas (Stage):** El MDD, el semáforo (`Status`), `precisionScore` y `Estimation` viven en **Prisma `Stage`** (no en `Project`). El API aplanó esos campos en `GET/PATCH /projects` para no romper el front; `PATCH` puede llevar `stageId`. Falkor SDD se ingiere por `stageId` (nodos `Stage`, `MDD_Section`, `DB_Entity`, `API_Endpoint`). Documentación: [STAGE-SDD.md](STAGE-SDD.md), [THEFORGE-INDEX.md §7](THEFORGE-INDEX.md), [MCP-ARQUITECTURA-THEFORGE.md](MCP-ARQUITECTURA-THEFORGE.md).

**Referencias:** [THEFORGE-INDEX.md](THEFORGE-INDEX.md), [../../mdd.md](../../mdd.md), [../../blueprint.md](../../blueprint.md).

## 4. Errores y soluciones

### Inferido del repo

- **Markdown libre vs estructurado:** Los agentes escribían markdown libre y luego se parseaba/sanitizaba → muchos edge cases y parsers frágiles. **Solución:** Estado central `mddStructured` (Zod), cada agente devuelve su slice; markdown siempre generado por `mddStructuredToMarkdown` (ver [PLAN-MDD-STRUCTURED-OUTPUT-JSON2MD.md](archive/PLAN-MDD-STRUCTURED-OUTPUT-JSON2MD.md)). Fallback: si un agente devuelve texto, extraer JSON y validar con Zod; si falla, poner ese texto en la sección como string para no romper el flujo.
- **Redactor:** Antes un agente "Redactor" reescribía todo el markdown. **Solución:** Eliminado; documento unificado por merge de slices en `mddStructured` + render; opcionalmente un "unificador de tono" sobre el objeto más adelante.

- **Spec no se persistía al confirmar "sí ingrésalo":** El chat respondía "El documento Spec ha sido actualizado con éxito" pero el Spec no se guardaba. **Causa:** El sistema solo persiste cuando la respuesta del LLM incluye el documento completo seguido del delimitador `---FIN_SPEC---`. El modelo no tenía instrucción explícita de devolver el documento en ese caso. **Solución:** En `ai.service.ts` se añadió para la pestaña Spec una instrucción explícita: cuando el usuario confirme integración (ej. "sí ingrésalo", "integralo", "aplica"), el modelo **debe** devolver el Spec completo actualizado y terminar con `---FIN_SPEC---`. Además se inyecta el Spec actual en el contexto (`currentSpecContent`) cuando la pestaña es Spec, y el front envía `specContent` en el body del chat para que el modelo pueda fusionar la nueva sección.
