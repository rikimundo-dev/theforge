| **WorkshopView.tsx** | Vista principal: **escritorio (`lg+`)** grid 3 columnas (chat \| documentos \| semáforo/costos). **Móvil:** una columna + barra inferior **Chat / Docs / Estado** (`mobileWorkshopColumn`), `100dvh` y `safe-area` en la navegación; header apilado y targets táctiles ~44px. **Header LEGACY:** si existe `project.theforgeProjectId`, se muestra bajo el bloque del título como `MCP <uuid>` (tooltip: id guardado; la API normaliza frente a `list_known_projects` → `roots[].id` + `scope` en ask/plan). **Barra de tabs (renglón 2):** en `sm` el hint de flujo y los botones van en columna; texto con `min-w-0 flex-1`; grupo de acciones `shrink-0`; **MDD Inicial** usa etiqueta corta en móvil (“doc. partida”) y la larga desde `sm`. Resto: ChatContainer (**Re-Valorar** → `reassess-complexity` + primer mensaje de entrevista); tabs de documentos; semáforo + costos + auditoría. **Semáforo / estimación:** `liveMetrics.readinessHints` bajo “Ver logs”; equipo con `ROLE_LABELS_ES` y roles dinámicos (no solo arquitecto/back/front); nómina vs mercado alineadas a `@theforge/business-rules`. **Banner `ComplexityPendingBanner`:** si hay propuesta HITL (`complexityPending`), confirmar/descartar o por chat. Header: selector **vista en vivo (etapa)** desde **`workshopStore.workshopStages`** + **Nueva etapa** (modal: nombre + **copiar MDD desde** cualquier etapa o vacío → `createWorkshopStage` con `copyMddFromStageId`). **Pestañas BRD / To-Be:** tabs propias (antes de MDD en la barra) con **`BrdTobeStagePanel`** `panel="brd"` \| `tobe` y el mismo chat por `activeTab` que Spec/MDD. **Pestaña MDD:** solo **`BrdTobeStagePanel`** `panel="gate-only"` (toggle exigir BRD+To-Be aprobados + estado de aprobación). Panel **Conformance**: línea **Blueprint vs MDD §3**; generación de **Contratos API** deshabilitada en UI si hay gaps §3. **Ayuda** → `WorkshopHelpModal` (`content/workshop-manual.md`); **Descargar todo (ZIP)**. **Paso 0:** Benchmark stream → `dbgaContent` + `complexityPending`; Deep Research → `phase0SummaryContent`. **`complexity === LOW`:** oculta MDD, Blueprint, API; **Generar entregables** → `generateDeliverablesCascade`. **`complexity === MEDIUM`:** `isTabVisibleForComplexity` (NEW/LEGACY). **Legacy:** Modificación + **MDD Inicial** (`legacyGenerateCodebaseDoc` con radios **Modo ingest** en `<details>` colapsable por defecto: `default` / `evidence_first` / `raw_evidence` → body del POST). Callout en Modificación si falta `codebaseDoc`; mensaje de error si el MCP no está configurado. Zustand. |
