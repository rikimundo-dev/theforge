# views

Vistas de la aplicación web.

| Componente           | Uso                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **LoginView.tsx**    | Login OTP **multi-usuario**: el correo se pide en el formulario y se envía al backend (`POST /auth/otp/request`) que valida contra la tabla `User` y manda el código por SMTP. Después se ingresa el código de 6 dígitos (`POST /auth/otp/verify`) en un **único** campo con `autocomplete="one-time-code"`, email oculto `autocomplete="username"` para autofill iOS/macOS, normalización de pegado (`123 456`, `123-456`) vía `utils/otpCode.ts`. Soporta SSO opcional vía `VITE_SSO_URL`. JWT en `utils/apiClient`. |
| **SetupView.tsx**    | Onboarding del primer usuario (cuando `GET /auth/has-users` devuelve `false`). Crea el primer `User` con rol `super_admin` y `mcpSecret` autogenerado vía `POST /auth/register-first-admin`.                                                                                                                        |
| **UsersView.tsx**    | Vista (solo admin) con `UsersList` en un `Card`, dentro del layout con sidebar. Entrada desde `App.tsx` cuando `usersViewOpen` es true; se sale con **Proyectos** en el menú o saliendo del taller.                                                                                                                      |
| **SettingsView.tsx** | Ajustes: instancias de proveedor IA (incl. **Modelo de auditor** por instancia), Secret MCP, Ariadne, toggle costo IA en semáforo.                                                                                                                               |
| **WorkshopView.tsx** | Vista principal (legacy: **Regenerar** en pestaña **MDD** → `legacy/generate-mdd` desde `codebaseDoc` / etapa; **MDD Inicial** → `generate-codebase-doc` vía Ariadne → `codebaseDoc`): **escritorio (`lg+`)** grid 3 columnas (chat \| documentos \| semáforo/costos). **Móvil:** una columna + barra inferior **Chat / Docs / Estado** (`mobileWorkshopColumn`), `100dvh` y `safe-area` en la navegación; header apilado y targets táctiles ~44px. **Header LEGACY:** si existe `project.theforgeProjectId`, se muestra bajo el bloque del título como `MCP <uuid>` (tooltip: id guardado; la API normaliza frente a `list_known_projects` → `roots[].id` + `scope` en ask/plan). **Barra de tabs (renglón 2):** en `sm` el hint de flujo y los botones van en columna; texto con `min-w-0 flex-1`; grupo de acciones `shrink-0`; **MDD Inicial** usa etiqueta corta en móvil (“doc. partida”) y la larga desde `sm`. Resto: ChatContainer (**Re-Valorar** → `reassess-complexity` + primer mensaje de entrevista); tabs de documentos; semáforo + costos + auditoría. **Semáforo / estimación:** `liveMetrics.readinessHints` bajo “Ver logs”; equipo con `ROLE_LABELS_ES` y roles dinámicos (no solo arquitecto/back/front); nómina vs mercado alineadas a `@theforge/business-rules`. **Banner `ComplexityPendingBanner`:** si hay propuesta HITL (`complexityPending`), confirmar/descartar o por chat. Header: selector **vista en vivo (etapa)** desde **`workshopStore.workshopStages`** + **Nueva etapa** (modal: nombre + **copiar MDD desde** cualquier etapa o vacío → `createWorkshopStage` con `copyMddFromStageId`). **Export SDD spec-kit** (icono carpeta): `GET /projects/:id/export/sdd-bundle` → ZIP `.specify/` + `specs/`. **Pestaña Tasks:** **Converge** (`POST /converge`) y **Tasks → GitHub Issues** (`POST /tasks-to-issues`). **Pestañas BRD / To-Be:** tabs propias (antes de MDD en la barra) con **`BrdTobeStagePanel`** `panel="brd"` \| `tobe` y el mismo chat por `activeTab` que Spec/MDD. **Pestaña MDD:** solo **`BrdTobeStagePanel`** `panel="gate-only"` (toggle exigir BRD+To-Be aprobados + estado de aprobación). Panel **Conformance**: línea **Blueprint vs MDD §3**; generación de **Contratos API** deshabilitada en UI si hay gaps §3. **Ayuda** → `WorkshopHelpModal` (`content/workshop-manual.md`); **Descargar todo (ZIP)**; ZIP gobernanza incluye bundle spec-kit para handoff. **Paso 0:** Benchmark stream → `dbgaContent` + `complexityPending`; Deep Research → `phase0SummaryContent`. **`complexity === LOW`:** oculta MDD, Blueprint, API; **Generar entregables** → `generateDeliverablesCascade`. **`complexity === MEDIUM`:** `isTabVisibleForComplexity` (NEW/LEGACY). **Legacy:** Modificación + **MDD Inicial** (`legacyGenerateCodebaseDoc` con radios **Modo ingest** en `<details>` colapsable por defecto: `default` / `evidence_first` / `raw_evidence` → body del POST). Callout en Modificación si falta `codebaseDoc`; mensaje de error si el MCP no está configurado. Zustand. |

Se entra desde la lista de proyectos en App; se pasa `projectId`, `projectName` y `onBack`.
