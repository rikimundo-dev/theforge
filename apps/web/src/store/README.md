| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **workshopStore.ts** | **`legacyGenerateCodebaseDoc(projectId, { responseMode })`** — POST `…/legacy/generate-codebase-doc` con body JSON (`responseMode` alineado con Ariadne `ask_codebase`). Sin segundo argumento, body `{}` (comportamiento API por env). **`setProjectRequireBrdTobeGate(projectId, bool)`** — PATCH `…/projects/:id` con `requireBrdTobeGate` (control usuario, no env). **`patchWorkshopStage(stageId, body)`** — PATCH `…/projects/:id/stages/:stageId` (BRD, To-Be, As-Is, `approveBrd` / `approveToBe`). **`legacyGenerateAsIsManual`**, **`legacySuggestBrdTobeFromCodebaseDoc`** — POST legacy As-Is / sugerencia BRD+To-Be desde `codebaseDoc`. **`suggestBrdTobeFromDbga(projectId, { stageId? })`** — POST `…/projects/:id/suggest-brd-tobe-from-dbga` (greenfield, `dbgaContent`; `loadingReason` `brd-tobe-from-dbga`). Streams MDD: **`blocked`** si el proyecto tiene `requireBrdTobeGate` y falta aprobación. Todas las llamadas al backend usan **`apiFetch`** (`utils/apiClient`) con JWT. Exporta **`sessionMessageBody`** para `POST /sessions/:id/messages` con `stageId` opcional e **`images`** opcionales en mensajes user. Store del Workshop: project (incluye **`stages[]`**, **`complexityPending`** HITL), **`workshopStages`** (copia sincronizada de `stages[]` para el selector multi-etapa), **`activeStageId`** (vista en vivo MDD), session, mddContent, loading, synced, error, **agentProgress**, **evaluatorCritique**. **`LiveMetricsResult`**: `roles` / `rolesHours` como `Record<string, number>`, **`readinessHints`** opcional (pistas IA/cascadas desde el estimador). **`conformance.blueprintDataModel`**: alineación Blueprint vs **§3 modelo de datos** del MDD; si `ok === false`, **`generateApiContracts`** no llama al API (y el backend también devuelve 400). **`confirmComplexityProposal` / `dismissComplexityProposal`** para el banner de complejidad. Tras stream DBGA (`/ai-analysis/stream`), si el `done` incluye `complexityProposal`, se actualiza el proyecto en memoria antes de `fetchProject`. **`clearChat`** borra también `evaluatorCritique` tras un borrado de historial exitoso. **`setActiveStageId`** usa `workshopStages` o `project.stages`, refresca `managerThreadId` vía `GET /ai-analysis/mdd/thread?stageId=` y **`fetchEstimation`**; **`createWorkshopStage`** (POST `/projects/:id/stages`) interpreta **`{ stage }`**, fusiona en `workshopStages` y foco en la nueva etapa (fallback `fetchProject` si el formato no es el esperado). **`fetchWelcome`** envía `stageId`; **`persistMddContent`** y **`fetchEstimation`** envían `stageId` cuando hay etapa activa; **`sendMessage`** incluye `stageId` en el body del orquestador y **`images`** en `POST /ai-orchestrator/chat/stream` y en **MDD Manager / resume** cuando el usuario adjunta; estado **`streamingUserImages`** durante el stream; **MDD stream** incluyen `stageId` cuando aplica. **Conformance** se actualiza al cargar proyecto (`fetchConformance`) y tras generar entregables. **`generateDeliverablesCascade`** → `POST /projects/:id/generate-deliverables`; si el API devuelve `{ queued, jobId }` (BullMQ + `REDIS_URL`), hace **polling** a `GET …/deliverables-jobs/:jobId` y actualiza `agentProgress` hasta `completed`, luego **`fetchProject` + `await fetchEstimation`** para refrescar semáforo y MXN. **Legacy entregables:** **`legacyGenerateDeliverables`** lee **`lastDeliverablesDebug`** del JSON de respuesta y lo guarda en **`lastLegacyDeliverablesDebug`**; en DEV hace **`console.debug('[LegacyDeliverables]', …)`**. Incluye **`sectionMergeTraces`** (ventanas § MDD + verify) cuando el API activa section merge. **`setProject`** hidrata **`lastLegacyDeliverablesDebug`** desde **`project.legacyFlowState.lastDeliverablesDebug`** (persistido en servidor). **`clearLegacyDeliverablesDebug`** limpia la traza en cliente. |
