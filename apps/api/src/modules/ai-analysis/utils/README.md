# Utils (ai-analysis)

Utilidades compartidas entre nodos MDD y Benchmark para evitar código duplicado.

- **parse-json.ts** – `parseJsonOrThrow(text, schema)`: parsea texto que puede contener JSON con o sin markdown code fence; usado por nodos MDD (manager, clarifier, security, integration, auditor). El agente Benchmark (scout, etc.) puede migrar a este util en el futuro para homologar.
- **mdd-sanitize.ts** – `normalizeMddFormat(draft)`: estandariza títulos (## 2. Modelo de datos, ## 3. Contratos de API), elimina líneas huérfanas con solo un número, envuelve SQL en ```sql si viene en texto plano, formatea JSON en Contratos de API (```json + pretty-print), e inserta `---` antes de cada `##` para separación visual (`ensureHorizontalRuleBeforeH2`). Corrige sección 2: SQL no cerrado antes de ### Diagrama / ```mermaid y encabezado pegado "### Diagrama entidad-relaciónmermaid" (`fixSection2UnclosedSqlAndGluedMermaid`). En Seguridad/Integración convierte JSON con `"heading"` o `"section"` + `"details"` a markdown (`sanitizeSeguridadIntegracionRawJson`, `convertSectionDetailsJsonToMarkdown`). Además descontamina el formato "bullet list as JSON lines" (cuando cada línea es una viñeta con un fragmento de JSON: `- {`, `- "title": ...`, `- "content": [`), reconstruyendo el JSON y convirtiéndolo a markdown legible (`unbulletAndParseSectionJson`, `objectWithContentArrayToMarkdown`). `jsonSectionToMarkdown`, `sanitizeContextSection`, `replaceContextWhenOnlyMetadata`, `preserveContextSectionIfSubstantial`, `ensureContratosSection`. Usado por nodos MDD y servicio al emitir markdown. Tests: `mdd-sanitize.spec.ts`.
- **suggest-mdd-governance-patterns.util.ts** – Preselección de ids del wizard SSOT desde DBGA, resumen Fase 0 y BRD (LLM + heurística).
- **mdd-diagram-suggestions.ts** – `suggestMddDiagrams(draft)`: analiza el MDD y devuelve sugerencias de diagramas Mermaid a insertar (erDiagram en modelo de datos si hay CREATE TABLE; stateDiagram-v2 en contratos si hay auth/login; flowchart en frontend si hay componentes). `injectMddDiagrams(draft, suggestions)` inserta los bloques en cada sección. `regenerateErDiagramFromSql(draft)`: regenera solo el erDiagram de la sección 2 desde el SQL (reemplaza o inserta). Usado por el nodo `mdd-diagram-injector`, por la tool `suggest_mdd_diagrams` y por el comando del Manager "regenerar diagrama ER".
- **mdd-pre-render.ts** – Re-export desde `engine/mdd-pre-render.ts`. `preRenderMddSanity(draft)`: valida Mermaid (sin coma PK/FK, espacios normalizados) y tablas markdown en §4 (alineación segunda fila, sin líneas en blanco). Devuelve `{ ok, code?, message? }`; códigos `ERR_MERMAID_SYNTAX`, `ERR_TABLE_SYNTAX`. Usado en `projects.service` antes de persistir `mddContent`.
- **mdd-complexity-rigor.ts** – Apéndices de prompt según `mddComplexity` (LOW/MEDIUM/HIGH): `clarifierComplexityAppendix`, `softwareArchitectComplexityAppendix`, `auditorConstitutionRigorAppendix`, `contextSynthesizerComplexityAppendix` (regenerar §1). Alineado con la relajación de gaps en `EstimationService` para el mismo nivel.
